## 版本 v1.1 - 饥饿惩罚重做 & 和平结局机制

**日期**：2026-04-16

### 改动背景

用户对游戏规则提出了以下反馈：
1. 死亡应该更重要，惩罚需要更重
2. 饥饿状态应该有更明显的效果
3. 需要增加游戏结束条件，特别是"和平结局"
4. Buff机制需要加强

### 改动详情

#### 1. 生命值系统重做

| 常量 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| HEALTH_INITIAL | 100 | 90 | 降低10点，营造"容易死"的环境 |
| HEALTH_MAX | 100 | 90 | 新增上限常量 |

**影响**：饥饿状态下从约20回合死亡缩短到约4.5回合

#### 2. 饥饿惩罚重做

| 改动 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| 饥饿扣血 | 5点/回合 | 20点/回合 | 大幅提高饥饿惩罚 |
| 饥饿时饱食消耗 | ×1.5 | 不变 | 删除了双重惩罚 |
| 饱食回血 | 无 | 5点/回合 | 新增，只要不是太饿就能回血 |
| 饱餐Buff回血 | 无 | 10点/回合 | 饱食>100时额外回血 |

**新常量**：
```java
HEALTH_REGEN_SATIETY_THRESHOLD = 30  // 回血阈值
HEALTH_REGEN_NORMAL = 5              // 普通回血量
HEALTH_REGEN_BUFF = 10               // Buff回血量
```

#### 3. Buff机制调整

| 常量 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| SATIETY_BUFF_RECOVERY_MULTIPLIER | 0.7 | 0.588 | 恢复加速从1.43倍提高到1.7倍 |

**效果对比**：
- 旧：休息恢复20/0.7 ≈ 28点（加速43%）
- 新：休息恢复20/0.588 ≈ 34点（加速70%）

#### 4. 白名单动作新增

新增 `provoke`（挑衅）动作：
```java
ALLOWED_ACTIONS = Set.of(
    "move", "eat", "rest", "talk", "trade", "provoke"
);
```

#### 5. AI判官规则有效期调整

| 常量 | 旧值 | 新值 | 说明 |
|------|------|------|------|
| CUSTOM_RULE_EXPIRE_TICKS | 20 | 8 | 快速变换规则，增加趣味 |

#### 6. 新增和平结局系统

**触发条件**：
1. 游戏进行至少40回合
2. 所有阵营都有Agent存活
3. 守序阵营持有"秩序之剑"
4. 守序阵营在公共频道发布"秩序宣言"
5. 激进阵营投票同意（过半数）

**新增常量**：
```java
PEACE_ENDING_MIN_TICKS = 40          // 和平结局最小回合数
ORDER_SWORD_SPAWN_TICK = 40          // 秩序之剑生成回合
MAX_GAME_TICKS = 50                  // 最大游戏回合数
ORDER_FACTION_BUFF = 0.1             // 守序阵营全员Buff
ORDER_DECLARATION_COOLDOWN = 10      // 秩序宣言冷却
```

#### 7. 新增秩序之剑系统

**设计**：
- 游戏进行到第40回合时，在随机位置生成
- 守序阵营持有：全员获得10%属性加成 + 可发布秩序宣言
- 其他阵营持有：无特殊效果（但可阻止守序阵营获得）
- 所有Agent都知道秩序之剑的作用

#### 8. 新增地图可视化文档

创建 `docs/map.md`，包含：
- 整体ASCII地图
- 节点详解
- 连接关系图
- 移动路线示例
- 战略意义说明
