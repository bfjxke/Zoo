# 如何测试新的地图界面

## ⚠️ 重要：浏览器缓存问题

如果您之前访问过前端页面，浏览器可能缓存了旧版本。请按以下步骤操作：

---

## 方法1：强制刷新（推荐）

### Chrome / Edge
1. 打开 http://localhost:3001
2. 按 `Ctrl + Shift + R` 或 `Ctrl + F5`
3. 或者：右键点击刷新按钮 → 选择"硬性刷新"

### Firefox
1. 打开 http://localhost:3001
2. 按 `Ctrl + Shift + R`

### Safari
1. 按 `Cmd + Option + R`

---

## 方法2：使用无痕模式

### Chrome 无痕窗口
1. 按 `Ctrl + Shift + N`
2. 访问 http://localhost:3001

### Edge InPrivate
1. 按 `Ctrl + Shift + P`
2. 访问 http://localhost:3001

---

## 方法3：清除浏览器缓存

### Chrome
1. 按 `F12` 打开开发者工具
2. **右键点击**刷新按钮
3. 选择"清空缓存并硬性重新加载"

---

## ✅ 验证新界面

打开浏览器后，您应该看到：

### 1. 地图区域变化
- ✅ **7个玻璃态节点卡片**（不再是简单的圆形）
- ✅ **圆角矩形**设计
- ✅ **半透明**效果
- ✅ **渐变边框**（根据阵营着色）
- ✅ **模糊背景**效果

### 2. 连接线变化
- ✅ **渐变色**线条（从源节点色渐变到目标节点色）
- ✅ **发光效果**
- ✅ **虚线样式**

### 3. Agent显示变化
- ✅ **堆叠显示**（同节点的Agent围成圆形）
- ✅ **呼吸动画**（光晕脉动）
- ✅ **阵营颜色**（蓝色/红色/绿色）

---

## 🔍 如果仍然看不到变化

### 检查1：前端服务是否使用新代码
```bash
# 查看前端服务状态
# 应该看到类似输出：
# VITE v5.4.21  ready in 1581 ms
# Local:   http://localhost:3001/
```

### 检查2：查看浏览器控制台
1. 按 `F12` 打开开发者工具
2. 切换到 **Console** 标签
3. 刷新页面
4. 查看是否有错误信息

### 检查3：检查网络请求
1. 按 `F12` 打开开发者工具
2. 切换到 **Network** 标签
3. 刷新页面
4. 查看 `/api/graph` 请求是否成功
5. 应该返回包含 `nodeCount: 7` 的JSON

---

## 📊 预期的API响应

打开 http://localhost:8080/api/graph 应该看到：

```json
{
  "nodes": [...],  // 7个节点
  "edges": [...],   // 8条边
  "nodeCount": 7,
  "edgeCount": 8
}
```

## 🎯 完整测试步骤

### 步骤1：验证后端API
```bash
# 打开浏览器访问
http://localhost:8080/api/graph

# 应该看到JSON数据
```

### 步骤2：验证前端加载
1. 打开 http://localhost:3001
2. **强制刷新** (Ctrl+Shift+R)
3. 查看控制台是否有错误
4. 查看网络请求是否成功

### 步骤3：验证视觉效果
1. 查看地图区域是否显示7个节点
2. 节点是否显示为**卡片样式**
3. 是否有**玻璃态效果**（模糊、透明）
4. Agent是否在节点上显示
5. Agent是否有**呼吸动画**

---

## ❌ 如果出现问题

### 问题1：API返回404
**解决**：重启后端服务
```bash
# 在后端目录
taskkill /F /PID <端口8080的进程ID>
mvn spring-boot:run
```

### 问题2：前端显示空白
**解决**：重启前端服务
```bash
# 在前端目录
# 停止当前服务
# 重新启动
npm run dev
```

### 问题3：仍然是旧界面
**解决**：
1. 清除浏览器缓存
2. 打开无痕模式
3. 或者等待2分钟后自动更新
