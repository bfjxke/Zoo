# GuardianEye-IIoT 沙箱动物园 - 游戏规则白皮书

> 本文档是游戏的"规则宪法"，所有游戏逻辑必须严格遵循此文档。

---

## 一、时间系统

### 1.1 回合（Tick）

- **定义**：Tick是游戏的基本时间单位
- **时长**：每个Tick = 30秒真实时间
- **触发**：系统自动每30秒执行一次Tick结算
- **游戏结束条件**：当所有Agent死亡且无法复活时，游戏结束（或者设置最大Tick数）

### 1.2 Tick结算流程

每个Tick按以下顺序执行4个阶段：

```
阶段1：被动消耗
  → 所有存活的Agent每回合扣除耐力和饱食度

阶段2：死亡处理
  → 检查死亡的Agent是否到时复活

阶段3：AI决策
  → 调用Python调度器，让AI Agent做出决策

阶段4：状态保存
  → 保存游戏状态到数据库
```

## 二、Agent属性系统

### 2.1 三项基础属性

| 属性 | 含义 | 初始值 | 上限（含Buff） |
|------|------|--------|----------------|
| 耐力(Stamina) | Agent的体力，决定能执行多少动作 | 100 | 100 |
| 饱食度(Satiety) | Agent的饱饿程度，耗尽会扣健康 | 100 | 140 |
| 健康(Health) | Agent的生命值，归零则死亡 | 100 | 100 |

### 2.2 状态判定规则

| 状态 | 触发条件 | 效果 |
|------|----------|------|
| 疲劳(Fatigued) | 耐力 < 20 | 所有消耗×1.5，恢复效果降低 |
| 饥饿(Hungry) | 饱食度 < 30 | 所有消耗×1.5，每回合扣5健康 |
| 死亡(Dead) | 健康 ≤ 0 | 无法行动，等待复活 |

### 2.3 属性惩罚叠加规则

**重要**：疲劳和饥饿的惩罚可以叠加！

| 状态组合 | 耐力消耗倍率 | 饱食度消耗倍率 |
|----------|--------------|----------------|
| 正常 | ×1.0 | ×1.0 |
| 只有疲劳 | ×1.5 | ×1.5 |
| 只有饥饿 | ×1.5 | ×1.5 |
| 疲劳+饥饿 | ×2.25 | ×2.25 |

## 三、动作系统

### 3.1 白名单动作（无需AI判官审批）

以下是5个"预设合法"的动作，Agent可以直接执行：

#### 3.1.1 移动(Move)

- **消耗**：15点耐力（受状态惩罚影响）
- **限制**：
  - 目标节点必须在当前节点的相邻列表中
  - 耐力必须足够支付消耗
- **失败情况**：
  - 节点不相邻
  - 耐力不足

#### 3.1.2 进食(Eat)

- **恢复**：+30点饱食度
- **上限**：饱食度最高可到140（超量进食）
- **限制**：饱食度已达140时无法再吃
- **特殊效果**：当饱食度 > 100时，获得"饱餐Buff"

#### 3.1.3 休息(Rest)

- **恢复**：+20点耐力（受状态惩罚和Buff影响）
- **限制**：耐力已达100时无法再休息
- **计算公式**：
  ```
  实际恢复 = 20 / 状态惩罚 / 恢复惩罚
  例如：疲劳状态 + 有Buff = 20 / 1.5 / 0.7 ≈ 19点
  ```

#### 3.1.4 发言(Talk)

- **消耗**：无耐力消耗
- **频道**：
  - `public`：公开频道，所有人可见
  - `lawful_private`：守序阵营私聊
  - `aggressive_private`：强势阵营私聊
  - `neutral_private`：中立阵营私聊
- **限制**：阵营私聊必须站在阵营基地才能发言

#### 3.1.5 交易(Trade)

- **消耗**：30点耐力（受状态惩罚影响）
- **限制**：耐力必须足够

## 四、消耗与恢复详细表

### 4.1 被动消耗（每回合自动发生）

| 属性 | 正常状态 | 疲劳状态 | 饥饿状态 | 疲劳+饥饿 |
|------|----------|----------|----------|-----------|
| 耐力 | -10 | -15 | -15 | -22.5 ≈ -23 |
| 饱食度 | -5 | -7.5 ≈ -8 | -7.5 ≈ -8 | -11.25 ≈ -11 |

### 4.2 移动消耗

| 状态 | 消耗公式 | 实际消耗 |
|------|----------|----------|
| 正常 | 15 | 15 |
| 疲劳 | 15 × 1.5 | 22.5 ≈ 23 |
| 有Buff | 15 × 0.7 | 10.5 ≈ 11 |
| 疲劳+Buff | 15 × 1.5 × 0.7 | 15.75 ≈ 16 |

### 4.3 休息恢复

| 状态 | 恢复公式 | 实际恢复 |
|------|----------|----------|
| 正常 | 20 | 20 |
| 疲劳 | 20 / 1.5 | 13.3 ≈ 13 |
| 饥饿 | 20 / 1.5 | 13.3 ≈ 13 |
| 疲劳+饥饿 | 20 / (1.5×1.5) | 8.8 ≈ 9 |
| 有Buff | 20 / 0.7 | 28.5 ≈ 29 |
| 疲劳+Buff | 20 / 1.5 / 0.7 | 19 ≈ 19 |

### 4.4 饥饿扣血

| 条件 | 每回合扣健康 |
|------|--------------|
| 饱食度 < 30 | -5点 |

## 五、食物Buff系统（核心新增机制）

### 5.1 Buff触发条件

- **触发时机**：饱食度 > 100 时
- **触发动作**：执行进食(Eat)动作后

### 5.2 Buff效果

- **效果名称**：饱餐Buff
- **效果描述**：耐力恢复速度加快50%
- **恢复倍率**：0.7（原本恢复20点，变成约29点）

### 5.3 Buff持续机制

- Buff与饱食度绑定
- 当饱食度下降到 ≤ 100时，Buff消失
- 饱食度越高，恢复越快

### 5.4 食物策略设计

| 饱食度 | 状态 | 恢复效果 | 策略建议 |
|--------|------|----------|----------|
| 0-30 | 饥饿 | 很差 | 优先找食物 |
| 30-100 | 正常 | 正常 | 平衡发展 |
| 100-140 | 有Buff | 加速50% | 囤积食物的优势期 |

### 5.5 囤积食物的博弈效果

- **强者优势**：能持续维持饱食度>100的Agent，恢复更快，战斗力更强
- **弱者联合**：无法囤积食物的Agent被迫联合对抗强者
- **偷窃动机**：食物储存在营地，促使弱势方偷窃强势方的存粮

## 六、死亡与复活系统

### 6.1 死亡触发

当健康值 ≤ 0时，Agent进入死亡状态

### 6.2 死亡惩罚

- Agent无法执行任何动作
- 位置重置为阵营基地
- 等待复活倒计时

### 6.3 复活机制

| 参数 | 数值 |
|------|------|
| 复活所需回合 | 3 Tick |
| 属性保留 | 50%（50点/项） |
| 状态清除 | 疲劳/饥饿状态重置 |

### 6.4 复活流程

```
Tick 1：Agent死亡，deathTicksRemaining = 3
Tick 2：倒计时 -1，deathTicksRemaining = 2
Tick 3：倒计时 -1，deathTicksRemaining = 1
Tick 4：倒计时 -1，deathTicksRemaining = 0 → 复活！
```

## 七、地图系统

### 7.1 节点列表

| 节点ID | 节点名称 | 所属阵营/类型 |
|--------|----------|---------------|
| base_lawful | 守序基地 | 守序阵营 |
| base_aggressive | 强势基地 | 强势阵营 |
| base_neutral | 中立基地 | 中立阵营 |
| center | 中央节点 | 中立区域 |
| forest | 森林 | 中立区域 |
| river | 河流 | 中立区域 |
| mountain | 山地 | 中立区域 |

### 7.2 节点相邻关系

```
        [forest]-----+
          |          |
[base_lawful]---[center]---[base_aggressive]
                    |
        [river]-----+         [mountain]
          |                      |
      [base_neutral]--------[base_aggressive]（重复，实际只有mountain连接）
```

更清晰的表示：

```
base_lawful    <-> center, forest
base_aggressive <-> center, mountain
base_neutral    <-> center, river
center          <-> base_lawful, base_aggressive, base_neutral, forest, river, mountain
forest          <-> base_lawful, center
river           <-> base_neutral, center
mountain        <-> base_aggressive, center
```

### 7.3 地图设计理念

- **辐射状结构**：center连接所有节点，是必经之路
- **阵营后花园**：每个阵营有自己的野外节点
- **战略要冲**：center是必争之地

## 十一、常见场景计算

### 场景1：正常Agent的行动能力

```
初始耐力：100
每回合消耗：10
行动次数 ≈ 100 / 10 = 10回合
游戏时长 ≈ 10 × 30秒 = 5分钟
```

### 场景2：疲劳Agent的生存压力

```
耐力：19（触发疲劳）
每回合消耗：15（×1.5）
行动次数 ≈ 19 / 15 ≈ 1回合后耗尽
```

### 场景3：Buff叠加恢复效果

```
饱食度：120（触发Buff）
疲劳状态
休息恢复 = 20 / 1.5 / 0.7 ≈ 19点
（比正常20点少一点，因为状态惩罚抵消了部分Buff）
```

### 场景4：饥饿链式惩罚

```
饱食度：30（触发饥饿）
每回合：
  - 耐力消耗：15
  - 饱食度消耗：7.5
  - 健康扣减：5

如果不吃东西，约14回合后健康归零死亡
```

### 场景5：超量进食策略

```
当前饱食：90
目标：触发Buff（饱食>100）

需要吃：1份食物（+30 → 120）

Buff效果：
  - 休息恢复：29点（而非20点）
  - 持续到饱食降到100以下
```
