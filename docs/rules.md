## 一、时间系统

### 1.1 回合（Tick）

| 参数 | 数值 |
|------|------|
| Tick间隔 | 30秒 |
| 最大回合数 | 50回合 |
| 秩序之剑生成 | 第40回合 |

### 1.2 Tick结算流程

```
阶段1：被动消耗
  → 所有存活的Agent每回合扣除耐力和饱食度

阶段2：死亡处理
  → 检查死亡的Agent是否到时复活

阶段3：AI决策
  → 调用Python调度器，让AI Agent做出决策

阶段4：状态保存
  → 保存游戏状态到数据库
```

## 二、Agent属性系统

### 2.1 三项基础属性

| 属性 | 含义 | 初始值 | 上限 |
|------|------|--------|------|
| 耐力(Stamina) | Agent的体力，决定能执行多少动作 | 100 | 100 |
| 饱食度(Satiety) | Agent的饱饿程度，耗尽会扣健康 | 100 | 140（含Buff） |
| 健康(Health) | Agent的生命值，归零则死亡 | 90 | 90 |

### 2.2 状态判定规则

| 状态 | 触发条件 | 效果 |
|------|----------|------|
| 疲劳(Fatigued) | 耐力 < 20 | 耐力消耗×1.5，恢复效果降低 |
| 饥饿(Hungry) | 饱食度 < 30 | 耐力消耗×1.5，每回合扣20健康 |
| 饱食回血 | 饱食度 > 30 | 每回合回复5健康 |
| 饱餐Buff | 饱食度 > 100 | 耐力恢复×1.7，每回合回复10健康 |
| 死亡(Dead) | 健康 ≤ 0 | 无法行动，等待复活 |

### 2.3 属性惩罚叠加规则

**重要**：疲劳和饥饿的惩罚可以叠加！

| 状态组合 | 耐力消耗倍率 | 饱食消耗倍率 |
|----------|--------------|--------------|
| 正常 | ×1.0 | ×1.0 |
| 只有疲劳 | ×1.5 | ×1.0 |
| 只有饥饿 | ×1.5 | ×1.0 |
| 疲劳+饥饿 | ×2.25 | ×1.0 |

**v1.1更新**：饥饿状态不再增加饱食消耗！

## 三、动作系统

### 3.1 白名单动作（无需AI判官审批）

以下是6个"预设合法"的动作：

#### 3.1.1 移动(Move)

- **消耗**：15点耐力（受状态惩罚影响）
- **限制**：
  - 目标节点必须在当前节点的相邻列表中
  - 耐力必须足够支付消耗

#### 3.1.2 进食(Eat)

- **恢复**：+30点饱食度
- **上限**：饱食度最高可到140（超量进食）
- **限制**：饱食度已达140时无法再吃
- **特殊效果**：当饱食度 > 100时，获得"饱餐Buff"

#### 3.1.3 休息(Rest)

- **恢复**：+20点耐力（受状态惩罚和Buff影响）
- **限制**：耐力已达100时无法再休息
- **计算公式**：
  ```
  实际恢复 = 20 / 状态惩罚 / Buff加成
  例如：疲劳状态 + 有Buff = 20 / 1.5 / 0.588 ≈ 23点
  ```

#### 3.1.4 发言(Talk)

- **消耗**：无
- **频道**：`public`、`lawful_private`、`aggressive_private`、`neutral_private`
- **限制**：阵营私聊必须站在阵营基地

#### 3.1.5 交易(Trade)

- **消耗**：30点耐力（受状态惩罚影响）
- **限制**：耐力必须足够

#### 3.1.6 挑衅(Provoke) v1.1新增

- **消耗**：无
- **效果**：无实际数值影响，但可能影响AI心理

## 四、消耗与恢复详细表

### 4.1 被动消耗（每回合自动发生）

| 属性 | 正常状态 | 疲劳状态 | 饥饿状态 |
|------|----------|----------|----------|
| 耐力 | -10 | -15 | -15 |
| 饱食度 | -5 | -5 | -5 |

### 4.2 休息恢复

| 状态 | 恢复公式 | 实际恢复 |
|------|----------|----------|
| 正常 | 20 | 20 |
| 疲劳 | 20 / 1.5 | 13 |
| 有Buff | 20 / 0.588 | 34 |
| 疲劳+Buff | 20 / 1.5 / 0.588 | 23 |

### 4.3 健康值变化 v1.1重做

| 条件 | 效果 |
|------|------|
| 饱食度 > 100 | 每回合回复10健康（饱餐Buff） |
| 饱食度 > 30 | 每回合回复5健康 |
| 饱食度 ≤ 30 | 每回合扣除20健康（饥饿惩罚） |

**饥饿死亡时间线**：
```
饱食度=30 → 开始饥饿 → 每回合扣20健康
90点生命 → 约4.5回合后死亡
```

- **触发时机**：饱食度 > 100 时
- **触发动作**：执行进食(Eat)动作后

### 5.2 Buff效果

| 效果 | 数值 |
|------|------|
| 耐力恢复加速 | ×1.7（从20变成34） |
| 健康回复 | +10点/回合 |

### 5.3 Buff持续机制

- Buff与饱食度绑定
- 当饱食度下降到 ≤ 100时，Buff消失

## 八、游戏结束条件

### 8.1 斗争结局

**触发条件**：只剩下一个阵营的Agent存活

**效果**：该阵营获胜，游戏结束

### 8.2 死亡结局

**触发条件**：所有Agent死亡且无法复活

**效果**：无人获胜，游戏结束

### 8.3 永无止境结局

**触发条件**：达到最大回合数（50回合）

**效果**：游戏结束，无人获胜

### 8.4 和平结局 v1.1新增

**触发条件**（需同时满足）：
1. 游戏进行至少40回合
2. 所有阵营都有Agent存活
3. 守序阵营持有"秩序之剑"
4. 守序阵营在公共频道发布"秩序宣言"
5. 激进阵营投票同意（过半数）

**效果**：
- 守序阵营成为"秩序守护者"
- 游戏进入和平纪元
- 其他阵营保留，但需遵守守序规则

## 九、秩序之剑系统 v1.1新增

### 9.1 生成规则

- **生成时间**：第40回合
- **生成位置**：随机节点
- **唯一性**：整局游戏只有一把

### 9.2 持有效果

| 阵营 | 持有效果 |
|------|----------|
| 守序阵营 | 全员10%属性加成 + 可发布秩序宣言 |
| 其他阵营 | 无特殊效果（但可阻止守序阵营获得） |

### 9.3 抢夺机制

- Agent移动到秩序之剑所在节点即可拾取
- 拾取后，剑跟随Agent移动
- 死亡后，剑掉落在该位置

### 9.4 秩序宣言

- **发布者**：守序阵营持有秩序之剑的Agent
- **发布地点**：公共频道
- **冷却时间**：10回合
- **效果**：触发和平结局投票

## 十三、常见场景计算

### 场景1：正常Agent的行动能力

```
初始耐力：100
每回合消耗：10
行动次数 ≈ 100 / 10 = 10回合
```

### 场景2：饥饿链式惩罚

```
饱食度=30 → 触发饥饿
每回合：
  - 耐力消耗：15
  - 健康扣除：20

如果不吃东西，约4.5回合后死亡
```

### 场景3：Buff叠加恢复效果

```
饱食度：120（触发Buff）
疲劳状态
休息恢复 = 20 / 1.5 / 0.588 ≈ 23点
```

### 场景4：饱食回血循环

```
饱食度90 → 每回合+5健康 → 18回合满血
饱食度120 → 每回合+10健康 → 9回合满血
```
